Practical Skills Task research

This page shows research on the 8 most important skills (soft skills and hard skills) that will support me next year. I will descibe how and why they will be useful for life after school.

Approach

Web Development (HTML, CSS, JavaScript) — (Hard Skill)

Web development is building websites that actually function, not just look good. I chose this because I already build websites and understand structure, navigation, and how frontend connects to backend systems like Google Apps Script. This helps me prepare for software engineering because I already understand how real systems are built and connected.

Version Control (GitHub) — (Hard Skill)

Version control helps manage and track changes in code so nothing gets lost. I already use GitHub to upload projects and manage updates. This prepares me for university and real development environments where GitHub is essential.

UI / Visual Design — (Hard Skill)

UI design focuses on how websites look and how easy they are to use. I apply this when designing layouts, spacing, and flow in my own projects. It helps me build interfaces that are clear and user-friendly.

Debugging & Problem Solving — (Hard Skill)

Debugging is fixing issues when code breaks. I already do this regularly in my projects by testing, identifying errors, and improving solutions. This is essential because coding always involves solving problems.

Adaptability — (Soft Skill)

Adaptability is adjusting when things change or don’t go as planned. I’ve had to adapt while working with different tools and fixing broken systems. This will help me in university where things move fast and constantly change.

Self-Learning — (Soft Skill)

Self-learning is teaching yourself without relying on others. I’ve learned most of my coding skills independently through practice and research. This is important for software engineering because continuous learning is required.

Time Management — (Soft Skill)

Time management is organising tasks effectively. I balance school, coding, and projects, which helps me stay productive. This will be essential for handling university deadlines and workload.

Communication — (Soft Skill)

Communication is explaining ideas clearly. I use this when breaking down coding concepts and explaining my projects. It is important for teamwork and working with non-technical people.
