Practical Skills Task research

This page shows research on the 8 most important skills (soft skills and hard skills) that will support me next year. I will descibe how and why they will be useful for life after school.

Hard & Soft Skills

Hard Skills Overview

These are technical skills related to coding, design, and development.

Web Development (HTML, CSS, JavaScript) — (Hard Skill)

Web development is basically building websites and making them actually work, not just look nice. This matters because everything is online now, and if you can build websites, you can create real things people can use.

I chose this because I already do it. Like, I’ve built my own website, worked with layouts, navigation, forms, and even connected things to Google Apps Script. So I’m not starting from zero, I already understand how things connect.

For example, when I built my contact form, I had to make sure the frontend (HTML) connects to the backend (Apps Script), and even though it broke a few times, I fixed it.

Next year, this helps me because software engineering is literally built on this. I’ll already understand structure, which means I won’t be completely lost when things get harder.

Version Control (GitHub) — (Hard Skill)

Version control is basically managing your code properly so you don’t mess everything up or lose your work. It also lets you track changes and improve things over time.

This matters because in real development, you don’t just write code once — you constantly update and fix it.

I chose this because I’ve already started using GitHub to create repositories, upload projects, and manage my files. Even just committing changes is already part of that process.

For example, when I updated my website, I had to push new changes and make sure everything stayed organised instead of just rewriting everything again.

Next year, this helps because in university and real jobs, GitHub is used everywhere. So instead of learning it from scratch, I’ll already be comfortable with it.

UI / Visual Design — (Hard Skill)

UI design is about how a website or system looks and how easy it is to use. It’s not just making things “pretty,” it’s making them make sense.

This matters because even if your code works, if the design is bad, people won’t use it.

I chose this because I naturally focus on layout, structure, and how things look. Like with my portfolio and case study pages, I didn’t just throw things on the screen — I actually thought about spacing, sections, and flow.

For example, when I designed my case study section, I made sure it was easy to read and visually clear instead of just dumping information.

Next year, this helps because software isn’t just code — it’s also user experience. So having design thinking already puts me ahead.

Debugging & Problem Solving — (Hard Skill)

Debugging is basically fixing things when they don’t work, and problem-solving is figuring out how to get from “broken” to “working.”

This matters because in coding, things break all the time. If you can’t fix problems, you can’t build anything properly.

I chose this because I already do this constantly. My code doesn’t always work the first time, so I have to figure out what’s wrong and fix it.

For example, when my email system wasn’t sending, I had to test different things, check errors, and keep trying until it finally worked.

Next year, this helps because software engineering is basically problem-solving all day. So this is one of the most important skills I can have.

Soft Skills Overview

These are personal skills that help with learning, communication, and adapting.

Adaptability — (Soft Skill)

Adaptability is being able to adjust when things change or don’t go as planned.

This matters because after school, everything is different — especially in university where you don’t get spoon-fed information.

I chose this because I already have to adapt when working with code. Things break, tools change, and sometimes I have to completely switch my approach.

For example, when Google Apps Script didn’t work the way I expected, I had to change how I was doing things instead of just giving up.

Next year, this helps because I’ll be dealing with new systems, new people, and new expectations, so being adaptable will stop me from falling behind.

Self-Learning / Initiative — (Soft Skill)

Self-learning is when you teach yourself instead of waiting for someone else to explain everything.

This matters because in tech, things change fast, and you can’t rely on being taught everything step by step.

I chose this because I already learn things on my own. No one taught me how to build my website or connect scripts — I figured it out.

For example, I learned how to use Google Apps Script and integrate it into my site just by testing and researching.

Next year, this helps because university expects you to learn independently. So this skill will make that transition easier.

Time Management — (Soft Skill)

Time management is being able to organise your time and get things done without falling behind.

This matters because after school, no one is constantly checking on you, so if you don’t manage your time, you fall behind quickly.

I chose this because I have to balance school, coding, and projects, even if it’s not always perfect.

For example, I worked on my LO project, my website, and school tasks at the same time, which forced me to plan when to do what.

Next year, this helps because university has deadlines, assignments, and exams all at once, so managing time properly will be important.

Communication — (Soft Skill)

Communication is being able to explain your ideas clearly to other people.

This matters because even if you understand something, it doesn’t help if you can’t explain it.

I chose this because my LO project is literally about teaching someone how to code, which means I have to explain things clearly.

For example, when I created my website lesson, I had to break things down step by step so someone else could understand it.

Next year, this helps because in software development, you often work with people who aren’t developers, so you need to explain your work properly.
