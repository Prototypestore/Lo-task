Reflection

Before doing this task, I used to think developing a skill just meant understanding how something works and being able to explain it. But through actually working on coding a table, I realised that understanding is only the starting point. Real skill development comes from applying that knowledge, making mistakes, fixing them, and improving the final result over time.

I chose coding as my practical skill because it aligns directly with what I want to do after school. Next year, I plan to go to university and study software engineering or software development. This isn’t just because it sounds like a good career, but because I genuinely enjoy building things and solving problems. I don’t enjoy purely theory-based work where nothing is created. I prefer practical work where I can see a result, even if it starts off messy and improves over time.

When I started coding the table, the basics seemed simple. I quickly understood how tags like <table>, <tr>, and <td> worked. However, the challenge came when I had to make the table look structured and visually clear. My first version technically worked, but it didn’t look professional. The spacing was off, alignment wasn’t consistent, and overall it looked unrefined. This showed me that in coding, functionality is not enough—presentation and structure also matter.

One of the main challenges I faced was slowing down and being intentional. I tend to move quickly and assume I understand something after trying it once. But in this task, small changes like adjusting padding or alignment often affected other parts of the table. This forced me to test my work step by step and actually understand what each line of code was doing instead of just guessing. It also made me realise that debugging and problem-solving are essential parts of coding, not just optional steps.

Another important part of this process was explaining what I was doing. Since this task required me to demonstrate and teach the skill, I had to clearly explain each step. This helped me identify gaps in my understanding. If I couldn’t explain why I used a certain tag or style, then I didn’t fully understand it. This process strengthened my communication skills and made my thinking more structured.

This experience connects directly to the skills I will need after school. In software development, it is not enough to know theory—you need to apply it consistently and improve your work. The process I followed in this task reflects important hard skills like HTML structure, basic design, and debugging, as well as soft skills such as patience, self-learning, and attention to detail. These are all essential for succeeding in university and in a future career in tech.

Moving forward, I need to continue developing both my technical and personal skills. Technically, I want to go beyond basic HTML and improve my use of CSS and JavaScript to create more advanced and interactive designs. I also want to start using tools like version control to manage my work more effectively. On a personal level, I need to improve my consistency, avoid rushing, and continue practicing problem-solving without giving up when something doesn’t work immediately.

Overall, this task showed me that skill development is not about completing something once and moving on. It is about continuous improvement, learning from mistakes, and pushing work beyond the basic level. This mindset will be essential as I transition into life after school, where I will need to take responsibility for my own learning and development. If I apply the same approach going forward, I will be better prepared not only for university, but also for building real-world projects and eventually turning my skills into something meaningful.
